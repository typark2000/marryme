# SPEC.md — marryme MVP

## Product summary
`marryme`는 매일 하나씩 새로운 청혼 페이지 디자인을 공개하고 사용자가 직접 인터랙션을 체험할 수 있게 하는 정적 웹앱이다.

## Problem statement
하루에 하나씩 올라오는 장난스럽고 기억에 남는 청혼 웹페이지를 빠르게 공개하고 공유할 수 있는 MVP가 필요하다.

## Scope
### In scope (MVP)
- 오늘의 청혼 페이지 1개를 메인 화면에서 보여주기
- 장난스러운 인터랙션 1개 제공 (`거절 버튼이 쉽게 눌리지 않음`)
- 사용자가 즉시 브라우저에서 체험 가능
- GitHub Pages로 배포 가능
- README에 실행/배포/데모 접근 방법 문서화
- 기본 접근성 보완(모션 감소, 키보드/터치 사용자 대응 장치)

### Out of scope (for now)
- 관리자 업로드 CMS
- 백엔드/DB
- 사용자 계정
- 댓글/공유/분석 대시보드
- 날짜별 전체 아카이브 UI 완성
- 다국어 지원

## Users
- 링크를 눌러 오늘의 청혼 디자인을 체험하는 방문자
- 매일 새로운 디자인을 업로드/관리하는 제작자(ty)

## User stories
1. 방문자로서, 링크 하나로 오늘의 청혼 페이지를 바로 체험하고 싶다.
2. 방문자로서, 재미있는 인터랙션을 경험하고 결과 화면도 보고 싶다.
3. 모션이나 예측 불가능한 움직임이 불편한 사용자로서, 접근 가능한 방식으로 페이지를 이용하고 싶다.
4. 제작자로서, 정적 파일만 수정해서 빠르게 배포하고 싶다.

## Acceptance criteria
- 메인 페이지에서 오늘의 디자인 제목/설명/CTA가 보인다.
- `좋아` 버튼을 누르면 성공 상태가 렌더링된다.
- 기본 모드에서 `싫어` 버튼은 장난스럽게 회피한다.
- 접근성 보조 장치가 있어 키보드/감각 민감 사용자도 페이지를 사용할 수 있다.
- GitHub Pages 배포가 동작한다.
- README에 run/build/deploy/demo instructions가 있다.

## Constraints
- 정적 프로젝트로 유지한다.
- GitHub Pages 우선 사용
- 비밀정보/토큰/환경변수 없이 공개 저장소로 운영

## Exit criteria
- 필수 문서(`SPEC.md`, `UX.md`, `TASKS.md`, `REVIEW.md`)가 존재한다.
- MVP가 로컬에서 바로 열리고 GitHub Pages에 배포된다.
- 리뷰 결과와 잔여 이슈가 `REVIEW.md`에 정리된다.
